Scratch : fonctions et variables

Variables, blocs personnalises, algorithmes avec entrees et sorties

I. Variables

Definition

Une variable est un espace memoire dans lequel on stocke une valeur (nombre, texte). En Scratch, on cree une variable avec le menu « Variables » et on peut la modifier avec les blocs « mettre [variable] a ... » et « ajouter ... a [variable] ».

Exemple : compteur

Quand drapeau vert clique

mettre [compteur] a [0]

repeter [10] fois

ajouter [1] a [compteur]

dire (compteur)

II. Entrees utilisateur

Bloc « demander »

Le bloc « demander [...] et attendre » pose une question a l'utilisateur. La reponse est stockee dans le capteur « reponse ».

III. Blocs personnalises (fonctions)

Definition

Un bloc personnalise (ou fonction) est un groupe d'instructions auquel on donne un nom. On peut lui associer des parametres (entrees). Cela evite de repeter du code et rend le programme plus lisible.

Creation

Menu « Mes blocs », cliquer « Creer un bloc », nommer le bloc (par exemple « calculer aire »), ajouter des parametres si necessaire.

IV. Algorithmes fondamentaux

Structures

Trois structures de base : la sequence (instructions les unes apres les autres), la condition (si ... alors ... sinon ...), la boucle (repeter ... fois, repeter jusqu'a ...).

Simulateur : programme de calcul

Programme : prendre un nombre, le multiplier par 3, ajouter 7, multiplier par 2, soustraire 6 fois le nombre de depart.

Exercices

Exercice 1

Ecrire un programme Scratch qui demande un nombre a l'utilisateur, calcule son carre, et affiche le resultat.

Corrige

Quand drapeau vert clique

demander [Quel nombre ?] et attendre

mettre [resultat] a [(reponse) * (reponse)]

dire (regrouper [Le carre est ] et (resultat))

Exercice 2

Programme de calcul : choisir un nombre, lui ajouter 4, multiplier par le nombre de depart, ajouter 4. Tester avec $n = 3$ et $n = 5$, puis prouver que le resultat est toujours un carre parfait.

Corrige

$n = 3$ : $(3+4) \times 3 + 4 = 21 + 4 = 25 = 5^2$. $n = 5$ : $(5+4) \times 5 + 4 = 45 + 4 = 49 = 7^2$.

Preuve : $(n+4) \times n + 4 = n^2 + 4n + 4 = (n+2)^2$. Le resultat est toujours le carre de $(n+2)$.

Exercice 3

Ecrire un algorithme qui teste si un nombre entre par l'utilisateur est premier. Decrire les etapes en pseudo-code.

Corrige

1. Demander n. 2. Si $n \leq 1$, dire « non premier ». 3. Mettre diviseur a 2, premier a VRAI. 4. Tant que diviseur $\leq \sqrt{n}$ : si le reste de n / diviseur = 0, mettre premier a FAUX. Ajouter 1 a diviseur. 5. Si premier = VRAI, dire « premier », sinon dire « non premier ».

Exercice 4

Creer un bloc personnalise « perimetre rectangle (L) (l) » qui calcule et affiche $2 \times (L + l)$. Utiliser ce bloc pour afficher le perimetre d'un rectangle de 8 cm sur 5 cm.

Corrige

Definition du bloc : definir perimetre rectangle (L) (l)

dire (regrouper [Perimetre = ] et ((L + l) * 2))

Appel : perimetre rectangle (8) (5) affiche « Perimetre = 26 ».
