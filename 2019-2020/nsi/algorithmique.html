Algorithmique

Recherche, tri, complexité et algorithmes classiques

I. Qu'est-ce qu'un algorithme ?

Définition

Un algorithme est une suite finie d'instructions précises qui résout un problème donné. Le mot vient du mathématicien persan Al-Khwarizmi (IXe siècle).

Pour être correct, un algorithme doit être :

  • Fini : il se termine toujours.
  • Précis : chaque étape est claire et non ambiguë.
  • Correct : il donne le bon résultat.
Un algorithme du quotidien

Chercher un mot dans un dictionnaire papier, c'est un algorithme. Personne ne lit le dictionnaire page par page du début à la fin : on l'ouvre au milieu, on regarde si le mot est avant ou après, et on recommence dans la bonne moitié. On vient de décrire la recherche dichotomique sans le savoir.

II. Complexité algorithmique

Qu'est-ce que la complexité ?

Deux algorithmes peuvent résoudre le même problème, mais l'un peut être des milliers de fois plus rapide. La complexité mesure le nombre d'opérations qu'un algorithme effectue en fonction de la taille $n$ de l'entrée.

On utilise la notation $O(...)$ pour décrire l'ordre de grandeur.

Pourquoi c'est important ?

Imagine que tu cherches un nom dans un annuaire de 50 000 contacts. La recherche séquentielle (lire un par un) fait en moyenne 25 000 comparaisons. La dichotomie en fait au maximum $\log_2(50\,000) \approx 16$. Seize comparaisons au lieu de vingt-cinq mille. Choisir le bon algorithme, c'est la différence entre une seconde et une heure de calcul.

III. Algorithmes de recherche

1. Recherche séquentielle (ou linéaire)

Principe

On parcourt la liste du début à la fin. Dès qu'on trouve l'élément, on s'arrête. Si on arrive à la fin sans le trouver, c'est qu'il n'y est pas.

Complexité : $O(n)$ dans le pire cas (l'élément est absent ou en dernière position).

def recherche_sequentielle(liste, cible):
    """
    Cherche 'cible' dans 'liste'.
    Retourne l'indice si trouvé, -1 sinon.
    """
    for i in range(len(liste)):
        if liste[i] == cible:
            return i       # trouvé !

    return -1              # pas trouvé


# Exemple : chercher un prénom
prenoms = ["Alice", "Bob", "Clara", "David", "Emma"]

resultat = recherche_sequentielle(prenoms, "Clara")
print(resultat)   # affiche 2

resultat = recherche_sequentielle(prenoms, "Zoé")
print(resultat)   # affiche -1

2. Recherche dichotomique

Principe (la liste doit être triée !)

On compare la cible avec l'élément du milieu :

  • Si c'est lui : trouvé !
  • Si la cible est plus petite : on cherche dans la moitié gauche.
  • Si la cible est plus grande : on cherche dans la moitié droite.

À chaque étape, on divise par 2 la zone de recherche.

Complexité : $O(\log n)$. C'est spectaculaire : avec 1 million d'éléments, au plus 20 étapes suffisent.

def dichotomie(liste_triee, cible):
    """
    Cherche 'cible' dans 'liste_triee' (qui DOIT être triée).
    Retourne l'indice si trouvé, -1 sinon.
    """
    gauche = 0
    droite = len(liste_triee) - 1

    while gauche <= droite:
        milieu = (gauche + droite) // 2

        if liste_triee[milieu] == cible:
            return milieu          # trouvé !

        elif liste_triee[milieu] < cible:
            gauche = milieu + 1    # chercher à droite

        else:
            droite = milieu - 1    # chercher à gauche

    return -1                      # pas trouvé


# Exemple : chercher un nombre dans une liste triée
nombres = [3, 7, 12, 18, 25, 31, 42, 56, 73, 89]

print(dichotomie(nombres, 42))    # affiche 6
print(dichotomie(nombres, 50))    # affiche -1

IV. Algorithmes de tri

Pourquoi trier ?

Trier, c'est mettre de l'ordre. Les contacts dans ton téléphone sont triés par nom. Les résultats de Google sont triés par pertinence. Les notes sont triées pour calculer la médiane. Sans tri, la recherche dichotomique ne fonctionne pas. Trier est l'une des opérations les plus fréquentes en informatique.

1. Tri par sélection

Principe

À chaque étape, on cherche le plus petit élément parmi ceux qui restent et on le met à sa place. Comme quand on range des cartes : on cherche le plus petit, on le met en premier, puis le plus petit des restants, etc.

Complexité : $O(n^2)$ dans tous les cas.

def tri_selection(liste):
    """
    Trie la liste en place par sélection du minimum.
    """
    n = len(liste)

    for i in range(n):

        # Chercher l'indice du minimum dans liste[i:]
        indice_min = i

        for j in range(i + 1, n):
            if liste[j] < liste[indice_min]:
                indice_min = j

        # Échanger liste[i] et liste[indice_min]
        liste[i], liste[indice_min] = liste[indice_min], liste[i]

    return liste


# Exemple
notes = [14, 8, 17, 5, 12, 19, 3]
print(tri_selection(notes))
# affiche [3, 5, 8, 12, 14, 17, 19]

2. Tri à bulles

Principe

On parcourt la liste en comparant chaque paire d'éléments voisins. Si deux voisins sont dans le mauvais ordre, on les échange. On recommence jusqu'à ce que la liste soit triée. Les grands éléments "remontent" comme des bulles dans l'eau.

Complexité : $O(n^2)$ dans le pire cas.

def tri_bulles(liste):
    """
    Trie la liste en place par échanges successifs
    de voisins mal ordonnés.
    """
    n = len(liste)

    for i in range(n):
        for j in range(0, n - i - 1):

            # Si les deux voisins sont dans le mauvais ordre
            if liste[j] > liste[j + 1]:

                # On les échange
                liste[j], liste[j + 1] = liste[j + 1], liste[j]

    return liste


# Exemple
ages = [17, 15, 18, 14, 16]
print(tri_bulles(ages))
# affiche [14, 15, 16, 17, 18]

3. Tri par insertion

Principe

On prend chaque élément et on l'insère à la bonne place dans la partie déjà triée. Comme quand on range une main de cartes : on prend une carte et on la glisse à sa place parmi celles déjà rangées.

Complexité : $O(n^2)$ dans le pire cas, mais $O(n)$ si la liste est presque triée.

def tri_insertion(liste):
    """
    Trie la liste en place en insérant chaque
    élément à sa bonne place.
    """
    for i in range(1, len(liste)):

        cle = liste[i]      # élément à insérer
        j = i - 1

        # Décaler les éléments plus grands vers la droite
        while j >= 0 and liste[j] > cle:
            liste[j + 1] = liste[j]
            j -= 1

        # Insérer la clé à sa place
        liste[j + 1] = cle

    return liste


# Exemple
scores = [42, 15, 78, 23, 56]
print(tri_insertion(scores))
# affiche [15, 23, 42, 56, 78]

V. Algorithmes classiques

1. Trouver le maximum

def maximum(liste):
    """
    Retourne le plus grand élément de la liste.
    """
    maxi = liste[0]

    for valeur in liste:
        if valeur > maxi:
            maxi = valeur

    return maxi


# Exemple
temperatures = [22, 28, 31, 19, 25]
print(maximum(temperatures))   # affiche 31

2. Compter les occurrences

def compter(liste, cible):
    """
    Compte combien de fois 'cible' apparaît dans 'liste'.
    """
    compteur = 0

    for element in liste:
        if element == cible:
            compteur += 1

    return compteur


# Exemple
votes = ["A", "B", "A", "C", "A", "B", "A"]
print(compter(votes, "A"))   # affiche 4

3. Vérifier si un mot est un palindrome

def est_palindrome(mot):
    """
    Un palindrome se lit pareil à l'endroit et à l'envers.
    Exemples : kayak, radar, rotor
    """
    mot = mot.lower()

    for i in range(len(mot) // 2):

        # On compare la lettre i avec son miroir
        if mot[i] != mot[len(mot) - 1 - i]:
            return False

    return True


# Tests
print(est_palindrome("kayak"))    # True
print(est_palindrome("radar"))    # True
print(est_palindrome("python"))   # False

4. Algorithme d'Euclide (PGCD)

def pgcd(a, b):
    """
    Calcule le PGCD de a et b
    par divisions successives (algorithme d'Euclide).
    """
    while b != 0:
        a, b = b, a % b

    return a


# Exemples
print(pgcd(48, 18))    # affiche 6
print(pgcd(100, 75))   # affiche 25

VII. Corrigés

Corrigé exercice 1

Corrigé

1. Séquentielle : $O(n)$. Dichotomie : $O(\log n)$.

2. Séquentielle : 100 000 comparaisons au max. Dichotomie : $\log_2(100\,000) \approx 17$ comparaisons au max.

3. La dichotomie regarde l'élément du milieu et décide d'aller à gauche ou à droite. Cela suppose que les éléments sont ordonnés. Si la liste n'est pas triée, "aller à droite" ne garantit pas que la cible s'y trouve.

Corrigé exercice 2

Corrigé

1. Tri par sélection de [38, 12, 7, 25, 3] :

Étape 1 : min = 3 → [3, 12, 7, 25, 38]

Étape 2 : min parmi [12, 7, 25, 38] = 7 → [3, 7, 12, 25, 38]

Étape 3 : min parmi [12, 25, 38] = 12 → [3, 7, 12, 25, 38]

Étape 4 : min parmi [25, 38] = 25 → [3, 7, 12, 25, 38]. Terminé.

2. Tri à bulles : il faut 4 passes (une passe = un parcours complet de gauche à droite).

3. Le tri par insertion est le plus rapide sur les listes presque triées. Si la liste est déjà triée, il fait seulement $n-1$ comparaisons (complexité $O(n)$).

def compter_comparaisons_selection(n):
    """
    Retourne le nombre de comparaisons
    du tri par sélection sur une liste de taille n.
    """
    comparaisons = 0

    for i in range(n):
        for j in range(i + 1, n):
            comparaisons += 1

    return comparaisons


# Test
print(compter_comparaisons_selection(10))     #  45
print(compter_comparaisons_selection(100))    #  4950
print(compter_comparaisons_selection(1000))   #  499500

def moyenne(liste):
    """Calcule la moyenne des éléments."""
    total = 0

    for valeur in liste:
        total += valeur

    return total / len(liste)


def deuxieme_max(liste):
    """Retourne le 2e plus grand élément."""
    max1 = max(liste[0], liste[1])
    max2 = min(liste[0], liste[1])

    for i in range(2, len(liste)):
        if liste[i] > max1:
            max2 = max1
            max1 = liste[i]
        elif liste[i] > max2:
            max2 = liste[i]

    return max2


def inverser(liste):
    """Retourne la liste inversée."""
    resultat = []

    for i in range(len(liste) - 1, -1, -1):
        resultat.append(liste[i])

    return resultat


# Tests
print(moyenne([10, 15, 20]))            # 15.0
print(deuxieme_max([3, 9, 1, 7, 5]))   # 7
print(inverser([1, 2, 3, 4, 5]))       # [5, 4, 3, 2, 1]
