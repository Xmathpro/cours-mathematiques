I. Qu'est-ce qu'un algorithme ?

Définition

Un algorithme est une suite finie d'instructions précises permettant de résoudre un problème en un nombre fini d'étapes. Le mot vient du mathématicien persan Al-Khwarizmi (IXe siècle), dont le nom latinisé a donné "algorithme".

Un algorithme doit être : correct (donne le bon résultat), fini (termine toujours), précis (chaque étape est non ambiguë) et efficace (utilise peu de ressources).

À Maurice : la recette du riz frit mauricien

Une recette de cuisine est un algorithme. Voici la "version informatique" du riz frit mauricien :

def riz_frit_mauricien(riz_cuit, oignons, oeufs, sauce_soja): # Étape 1 : préconditions if riz_cuit == False: return "Erreur : il faut du riz cuit d'abord" # Étape 2 : frire les oignons for oignon in oignons: faire_frire(oignon, huile="huile végétale") # Étape 3 : ajouter les œufs melanger(oeufs, oignons) # Étape 4 : ajouter le riz et assaisonner ajouter(riz_cuit) assaisonner(sauce_soja) return "riz frit mauricien prêt 🍳"

Chaque étape est précise, l'ordre est important, et le résultat est bien défini. C'est un algorithme.

II. Complexité algorithmique

Définition

La complexité d'un algorithme mesure son coût en temps ou en mémoire en fonction de la taille de l'entrée $n$. On utilise la notation $O(...)$ (notation "grand O") pour décrire l'ordre de grandeur.

III. Algorithmes de recherche

1. Recherche séquentielle — $O(n)$

Principe

On parcourt la liste du début à la fin. Dans le pire cas, on fait $n$ comparaisons (élément absent ou en dernier).

def recherche_sequentielle(liste, cible): # Cherche cible dans liste, retourne son indice ou -1 for i in range(len(liste)): if liste[i] == cible: return i # trouvé à l'indice i return -1 # pas trouvé # Exemple mauricien : chercher une plage dans la liste plages = ["Belle Mare", "Flic en Flac", "Blue Bay", "Le Morne", "Trou aux Biches"] print(recherche_sequentielle(plages, "Blue Bay")) # → 2

🏖️ Recherche séquentielle : chercher une plage de Maurice

2. Recherche dichotomique — $O(\log n)$

Principe (liste triée obligatoire)

On compare l'élément cherché avec l'élément du milieu. Si c'est lui, on a trouvé. Sinon on sait dans quelle moitié chercher. On divise l'espace de recherche par 2 à chaque étape.

Avec 1 million d'éléments, la dichotomie trouve en au plus $\log_2(10^6) \approx 20$ comparaisons, contre 1 million pour la séquentielle !

def dichotomie(liste, cible): # Liste TRIÉE obligatoire gauche, droite = 0, len(liste) - 1 while gauche <= droite: milieu = (gauche + droite) // 2 if liste[milieu] == cible: return milieu elif liste[milieu] < cible: gauche = milieu + 1 # chercher à droite else: droite = milieu - 1 # chercher à gauche return -1 # Chercher une année parmi les prix Nobel mauriciens fictifs annees = [1968, 1972, 1985, 1991, 1999, 2005, 2010, 2015, 2018, 2023]

IV. Algorithmes de tri

Pourquoi trier ?

Trier des données est l'une des opérations les plus fréquentes en informatique : trier les clients d'une banque (MCB, SBM) par solde, les produits d'une boutique par prix, les élèves par note, les courses de taxi par distance. Un bon algorithme de tri peut faire économiser des heures de calcul sur de grandes bases de données.

1. Tri par sélection — $O(n^2)$

Principe

À chaque étape $i$, on cherche le minimum des éléments restants et on le place à la position $i$.

def tri_selection(liste): n = len(liste) for i in range(n): # Chercher le minimum dans liste[i:] idx_min = i for j in range(i + 1, n): if liste[j] < liste[idx_min]: idx_min = j # Échanger liste[i] et liste[idx_min] liste[i], liste[idx_min] = liste[idx_min], liste[i] return liste # Trier les prix (Rs) des fruits au marché de Port Louis prix = [45, 12, 78, 23, 56, 8, 34] print(tri_selection(prix)) # [8, 12, 23, 34, 45, 56, 78]

🛒 Tri par sélection — prix des légumes au marché de Port Louis (Rs)

2. Tri à bulles — $O(n^2)$

Principe

On parcourt la liste en comparant chaque paire d'éléments consécutifs. On échange si besoin. Les grands éléments "remontent" comme des bulles.

📊 Tri à bulles — classement des distances des villages de Maurice au centre (km)

3. Tri par insertion — $O(n^2)$ pire cas, $O(n)$ si presque trié

Principe

On insère chaque élément à sa bonne place dans la partie déjà triée, comme on insère une carte dans une main de jeu triée.

def tri_insertion(liste): for i in range(1, len(liste)): cle = liste[i] j = i - 1 while j >= 0 and liste[j] > cle: liste[j + 1] = liste[j] j -= 1 liste[j + 1] = cle return liste

def maximum(liste): # Trouver le salaire max parmi les employés (Rs) maxi = liste[0] for val in liste: if val > maxi: maxi = val return maxi salaires = [25000, 48000, 32000, 71000, 28000] print(maximum(salaires)) # 71000 Rs

def est_palindrome(chaine): # Un palindrome se lit pareil dans les deux sens chaine = chaine.lower().replace(" ", "") return chaine == chaine[::-1] print(est_palindrome("Laval")) # True print(est_palindrome("Mauritius")) # False print(est_palindrome("kayak")) # True

Corrigé exercice 3

Corrigé

1. $O(n^2)$ : si $n$ est multiplié par 10, le temps est multiplié par $100$. Donc 100 secondes pour $n=10\,000$.

2. $O(\log n)$ : $\log_2(10^6) \approx 20$ contre $\log_2(10^3) \approx 10$. Le temps double (environ 20 ms).

3. Avec 50 000 noms, la recherche séquentielle fait en moyenne 25 000 comparaisons. La dichotomie en fait au plus $\log_2(50000) \approx 16$. Sur un annuaire trié par ordre alphabétique, tout le monde fait naturellement de la dichotomie (on ouvre au milieu, puis on va à gauche ou à droite).
